[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture to see how it works under the hood. 
Every query starts at the Security Checkpoint node, which scrubs PII and checks for prompt injections or compliance blocks before any LLM processing occurs. 
If clean, it routes to the main Orchestrator agent. 
The Orchestrator acts as our coordinator, using AgentTools to delegate tasks to two specialized sub-agents: the Itinerary Planner for building travel schedules, and the Booking Coordinator for researching travel options. 
These sub-agents connect to a custom stdio MCP Server to fetch real-time weather forecasts, flight schedules, and hotel rates. 
If the Orchestrator prepares a booking, it triggers a routing decision to pause at the human-in-the-loop Approval Gate, requiring explicit consent before calling the final node to confirm the booking.

[3:00 — SECURITY & MCP]
Our security node is highly robust, utilizing regex patterns to scrub sensitive data, intercept prompt injection keywords, and output structured JSON audit logs directly to standard error. Our custom MCP server manages standard tools for querying weather, flights, and hotels using stdio transport parameters.
